Microsoft Visual Studio .NET Windows Component Update

Web Project Requirements

If you want to create Web projects on this machine, you must first install additional components. To create Web projects, both Internet Information Services (IIS) and FrontPage Server Extensions (FPSE) must be installed and configured. You can, however, create Web projects on remote machines that have these components installed and configured.

How do I install these components?

Return to the Web Project Requirements dialog box and click Install Components . A Web page will appear, listing the steps to install both components. After you have installed and configured Internet Information Services and FrontPage Server Extensions, you can return to setup by running Setup on CD1 or DVD of the Visual Studio product you are installing.

What happens if I choose not to install these components?

Visual Studio .NET setup will continue. However, you will be able to create Web projects only on remote servers with the appropriate components installed.  For more information on setting up a remote server, see the section "Configuring a Web Server for Visual Studio .NET" in the Visual Studio .NET Readme at the root of CD1 or DVD.

Can I install Internet Information Services after Visual Studio .NET setup?

Yes, however, this is not recommended. Visual Studio .NET setup will install the Microsoft .NET Framework, which registers new extensions with Internet Information Services. If you install Internet Information Services later, you will need to repair the Microsoft .NET Framework to register its extensions.  To repair the .NET Framework:

  1. Insert the Visual Studio .NET Windows Component Update CD or Visual Studio .NET DVD.
  2. On the Start menu, choose Run.
  3. Type the following:
    <CD Drive >:\dotNetFramework\dotnetfx.exe /t:c:\temp /c:"msiexec.exe /fvecms c:\temp\netfx.msi"

    —or—

    <DVD Drive >:\wcu\dotNetFramework\dotnetfx.exe /t:c:\temp /c:"msiexec.exe /fvecms c:\temp\netfx.msi"
